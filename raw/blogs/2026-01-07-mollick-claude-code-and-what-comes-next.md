# RAW — Claude Code and What Comes Next (Ethan Mollick, One Useful Thing)
Source: https://www.oneusefulthing.org/p/claude-code-and-what-comes-next · 2026-01-07 · captured 2026-06-22 · channel: blog

- Opening demo: gave Claude Code the prompt to "develop a web-based or software-based startup idea that will make me $1000 a month where you do all the work… i shouldn't have to do anything at all." Claude asked 3 multiple-choice questions, decided to sell sets of 500 prompts for $39, then worked independently FOR 1 HOUR 14 MINUTES, generating hundreds of code files and prompts, and produced a single file that deployed a working sales website (with "very sketchy fake marketing claims"). The Stripe payment link actually worked. He suspects it would actually make the promised $1,000.
- Frames Claude Code as "a sudden capability leap in AI in the past month or so." The leap is NOT one breakthrough but a **combination of two advances**: (1) the latest AIs do far more work autonomously while self-correcting errors (esp. in programming); (2) AIs are given an **"agentic harness"** of tools/approaches to solve problems new ways. (Explicitly back-references "Real AI Agents and Real Work.")
- Cites **METR** length-of-task chart again — exponential, with large leaps in recent months; "correlates with most other measures."
- Caveat: these tools are built FOR programmers (assume Python knowledge, "1980s computer lab" interfaces) — a shame, because they're "broadly useful to knowledge workers of all types." Focuses on Claude Code + Opus 4.5; notes competition = OpenAI Codex w/ GPT-5.2 and Google Antigravity w/ Gemini 3.
- Demonstrates Claude Code taking control of the **browser** to do user testing of the live site "from different personas," giving a report. Because AIs are **sycophantic**, he asked for a more critical version — which better nailed issues and spotted the fake reviews.
- "The Magic Tricks" — the agentic-harness mechanics:
  - **Compacting**: context windows (150k+ words) fill fast (whole conversation + every doc/image + system prompts); no real long-term memory. When Claude Code runs out of context it stops and "compacts" — takes notes on where it was, clears context, a fresh instance reads the notes and resumes (analogy: Memento's tattoos). This is why it can run for hours.
  - **Skills**: instructions the AI decides when to use, containing prompts AND the tools needed for a task (Website Creator skill, Excel skill, etc.). Solves the problem that long prompts eat context and require feeding the right prompt at the right time. Easy to create in plain language; the AI can help create them. Cites Jesse Vincent's free "superpowers" skills list (github.com/obra/superpowers) for a full software dev process. Shows Anthropic's Design Skill text — "written in plain language and trusts the AI to make decisions."
  - **Subagents**: launching other specialized AIs for specific problems; hand off easy tasks to cheaper/faster models; run many processes at once "like a team"; each can have its own context window. He built separate subagents for research and image creation.
  - **MCP (Model Context Protocol)**: companies expose tools/access to any AI — MCPs from publishers (scientific papers), payment companies (financial data), software providers, etc. "A very flexible system where a smart generalist AI like Claude Opus 4.5 can apply specialized skills on the fly."
- "Claude Code is particularly powerful because it works on your computer and your files." An AI that can do almost anything a human with access to your machine can — read/create files (PowerPoint/Word are "just code"), use the web via your browser, write/execute programs. Risks flagged: deleting files, executing code with unintended consequences, accessing sensitive browser data. "Make backups, use a dedicated folder."
- "An Amateur's Guide": Claude Desktop (as of "yesterday") gives an easier non-CLI way to use Claude Code; requires ≥$20/mo. Slash commands: /agents, /skills. Anecdote: while writing, he had Claude Code building a civilization-simulation game (plate tectonics, weather, ruler family trees, AI event narrator) — "Unlike previous vibe coding experiences, the AI never got stuck or went in circles."
- Closes with **Andrej Karpathy** quote: "I've never felt this much behind as a programmer. The profession is being dramatically refactored… I could be 10X more powerful if I just properly string together what has become available over the last ~year and a failure to claim the boost feels decidedly like skill issue." Mollick: "New harnesses that make AI work for other knowledge tasks are coming in the near future."

Stake: Wharton/UPenn associate professor; Co-Intelligence author. Adoption-optimist / hands-on practitioner. (Note: he often gets early access to frontier models — Anthropic, OpenAI, Google all court him.) Weight the harness mechanics and capability-jump observation; discount the optimism.
