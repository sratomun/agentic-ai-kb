## Key quotes (verbatim)
### The November 2025 inflection point
- "in November we had what I call the inflection point where GPT 5.1 and Claude Opus 4.5 came along… in a way that crossed a threshold where previously the code would mostly work, but you had to pay very close attention to it. And suddenly we went from that to… almost all of the time it does what you told it to do, which makes all of the difference in the world."

### Software engineers as bellwethers
- "I can churn out 10,000 lines of code in a day. And most of it works. Is that good? Like, how do we get from most of it works to all of it works?… which I think makes us a bellwether for other information workers."
- "Code is easier than almost every other problem that you pose these agents because code is obviously right or wrong… If it writes you an essay, if it prepares a lawsuit for you, it's so much harder to derive if it's actually done a good job… It came for us first."

### Responsible vibe coding
- "If you're vibe coding something for yourself, where the only person who gets hurt if it has bugs is you, go wild… The moment you ship your vibe coding code for other people to use, where your bugs might actually harm somebody else, that's when you need to take a step back."

### Dark Factories
- "probably 95% of the code that I produce, I didn't type myself. That world is practical already because the latest models are good enough… The next rule though, is nobody *reads* the code. And this is the thing which StrongDM started doing last year."

### Bottleneck has moved to testing / prototyping
- "It used to be, you'd come up with a spec and you hand it to your engineering team. And three weeks later… they'd come back with an implementation. And now that maybe takes three hours… Now, where else are the bottlenecks?"
- "A UI prototype is free now… anyone who's doing product design and isn't vibe coding little prototypes is missing out on the most powerful boost that we get in that step."
- "Throughout my entire career, my superpower has been prototyping… And that's gone. Anyone can do what I could do."

### Exhausting / careers
- "using coding agents well is taking every inch of my 25 years of experience as a software engineer, and it is mentally exhausting. I can fire up four agents in parallel… And by like 11 AM, I am wiped out for the day."
- "There's an element of sort of gambling and addiction to how we're using some of these tools."
- "I've got 25 years of experience in how long it takes to build something. And that's all completely gone—it doesn't work anymore."
- "the one thing AI can never have is agency because it doesn't have human motivations… invest in your own agency."
- People in the middle: "It's really good for experienced engineers… It's really good for new engineers… The problem is the people in the middle."

### Evaluating software
- "It turns out when I'm using somebody else's software, the thing I care most about is I want them to have used it for months."
- "I've got some very cool software that I built that I've *never used*. It was quicker to build it than to actually try and use it!"

### Misconception that AI tools are easy
- "That's one of the great misconceptions in AI is that using these tools effectively is easy. It takes a lot of practice…"

### Security research
- "In the past sort of three to six months, they've started being credible as security researchers, which is sending shockwaves through the security research industry."
- On junk reports: "ChatGPT can produce a very well formatted report of a vulnerability. It's a total waste of time. It's not actually verified as being a real problem." (Cites Anthropic/Firefox collaboration where Anthropic *verified* every issue.)

### OpenClaw
- "OpenClaw demonstrates that people want a personal digital assistant so much that they are willing to not just overlook the security side of things… hundreds of thousands of people got it set up."
- "The first line of code for OpenClaw was written on November the 25th. And then in the Super Bowl, there was an ad for AI.com… So we went from first line of code in November to Super Bowl ad in… Three and a half months."

### Pelican benchmark
- "There appears to be a very strong correlation between how good their drawing of a pelican riding a bicycle is and how good they are at everything else. And nobody can explain to me why that is."
