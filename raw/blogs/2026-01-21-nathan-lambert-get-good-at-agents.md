# RAW — Get Good at Agents (Nathan Lambert, Interconnects)
Source: https://www.interconnects.ai/p/get-good-at-agents · 2026-01-21 · captured 2026-06-22 · channel: blog

- Follow-up to "Claude Code Hits Different." Thesis: applying his last-few-years working habits to agents is "fundamentally wrong." Today's habits (micromanaging, tiring himself out, setting agents on too-small tasks) limit the uplift. Better mode: "more open ended, more ambitious, more asynchronous."
- "Agents push the humans up the org chart." Every engineer must learn to design systems; every researcher to run a lab. His role is shifting "more to pointing the army rather than using the power-tool."
- "Being good at using AI today is a better moat than working hard." / "Software is becoming free, good decision making in research, design, and product has never been so valuable."
- Default workflow: GPT-5 Pro for planning, Claude Code with Opus 4.5 for implementation; passes info back to GPT-5 Pro for deep search when stuck. Codex with GPT-5.2 (xhigh) "very capable, more meticulous than Claude even," but he hasn't figured out how to get the best out of it. "GPT Pro feels itself to be a strong agent trapped in the wrong UX."
- Leaves Claude Code instances running on his DGX Spark implementing features in their RL codebase while at dinner/work: "they make mistakes, they catch most of their own mistakes... but they're capable."
- The crucial open skill: "how do we stack these bricks into coherent long-term projects?" — moving agents from toy software to meaningful long-term tasks. "There are no hints or guides on working with agents at the frontier — the only way is to play with them." Advice: give an agent one of your hardest tasks, see what it gets stuck on.
- Cites a cluster of like-minded writers grappling with the moment: Import AI 441 (Jack Clark), Steve Newman (Hyperproductivity), Tim Dettmers (Use Agents or Be Left Behind?), Steve Yegge on vibe coding, Dean W. Ball (Among the Agents).
Stake: researcher / open-models advocate (transitioning out of Ai2 at time of writing); low commercial stake.
