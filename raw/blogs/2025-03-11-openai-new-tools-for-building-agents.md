# New tools for building agents

Author: OpenAI
Date: March 11, 2025
Category: Product
Captured: 2026-06-22
Vendor stake: OpenAI launching/marketing its own agent platform (Responses API, built-in tools, Agents SDK). Benchmark figures and customer quotes are self-reported.

---

Today, we're releasing the first set of building blocks that will help developers and enterprises build useful and reliable agents. We view agents as systems that independently accomplish tasks on behalf of users. Over the past year, we've introduced new model capabilities—such as advanced reasoning, multimodal interactions, and new safety techniques—that have laid the foundation for our models to handle the complex, multi-step tasks required to build agents. However, customers have shared that turning these capabilities into production-ready agents can be challenging, often requiring extensive prompt iteration and custom orchestration logic without sufficient visibility or built-in support.

To address these challenges, we're launching a new set of APIs and tools specifically designed to simplify the development of agentic applications:

- The new Responses API, combining the simplicity of the Chat Completions API with the tool use capabilities of the Assistants API for building agents
- Built-in tools including web search, file search, and computer use
- The new Agents SDK to orchestrate single-agent and multi-agent workflows
- Integrated observability tools to trace and inspect agent workflow execution

These new tools streamline core agent logic, orchestration, and interactions, making it significantly easier for developers to get started with building agents.

## Introducing the Responses API

The Responses API is our new API primitive for leveraging OpenAI's built-in tools to build agents. It combines the simplicity of Chat Completions with the tool-use capabilities of the Assistants API. With a single Responses API call, developers will be able to solve increasingly complex tasks using multiple tools and model turns. To start, the Responses API will support new built-in tools like web search, file search, and computer use. It brings usability improvements including a unified item-based design, simpler polymorphism, intuitive streaming events, and SDK helpers like `response.output_text`. The API is available to all developers starting today and is not charged separately—tokens and tools are billed at standard rates.

## What this means for existing APIs

- Chat Completions remains OpenAI's most widely adopted API and continues to be supported; developers who don't require built-in tools can keep using it. The Responses API is a superset of Chat Completions with the same performance; for new integrations OpenAI recommends starting with the Responses API.
- The Assistants API: improvements were incorporated into the Responses API. OpenAI plans to achieve full feature parity (Assistant-like and Thread-like objects, Code Interpreter), then formally announce deprecation of the Assistants API with a target sunset date in mid-2026, with a migration guide. "The Responses API represents the future direction for building agents on OpenAI."

## Built-in tools

### Web search
Developers can get fast, up-to-date answers with citations from the web. Available as a tool with gpt-4o and gpt-4o-mini, pairable with other tools or function calls. Web search in the API is powered by the same model used for ChatGPT search. On SimpleQA, GPT-4o search preview and GPT-4o mini search preview score 90% and 88% respectively. Example customer: Hebbia (asset managers, PE, credit, law). Direct access to fine-tuned search models in Chat Completions via `gpt-4o-search-preview` and `gpt-4o-mini-search-preview`. Pricing starts at $30 and $25 per thousand queries respectively.

### File search
Improved file search tool: multiple file types, query optimization, metadata filtering, custom reranking. Use cases: customer-support agent FAQs, legal assistant case reference, coding agent docs. Example customer: Navan (AI travel agent / RAG pipeline without extra tuning). Priced at $2.50 per thousand queries; storage $0.10/GB/day, first GB free. New search endpoint added to Vector Store API objects.

### Computer use
The computer use tool in the Responses API is powered by the same Computer-Using Agent (CUA) model that enables Operator. This research preview model set a new state-of-the-art record: 38.1% success on OSWorld for full computer use tasks, 58.1% on WebArena, and 87% on WebVoyager for web-based interactions. The tool captures mouse and keyboard actions generated by the model. Example customers: Unify (GTM intent/research), Luminai (legacy-system RPA replacement). Safety: prompt-injection safety checks, confirmation prompts for sensitive tasks, environment isolation tools, enhanced policy-violation detection; human oversight recommended (OSWorld at 38.1% indicates the model is not yet highly reliable for OS tasks). Available as a research preview in the Responses API for select developers in usage tiers 3-5. Priced at $3/1M input tokens and $12/1M output tokens.

Benchmark table (OpenAI CUA vs previous SOTA):
- OSWorld (computer use): OpenAI CUA 38.1% vs previous SOTA 22.0% (Anthropic); human 72.4%
- WebArena (browser use): OpenAI CUA 58.1% vs previous SOTA 36.2%; another browsing-agent SOTA 57.1%; human 78.2%
- WebVoyager: OpenAI CUA 87.0%; previous SOTA 56.0%; browsing-agent SOTA 87.0%

## Agents SDK

The new open-source Agents SDK simplifies orchestrating multi-agent workflows and improves on Swarm, an experimental SDK OpenAI released previously. Improvements:
- Agents: configurable LLMs with clear instructions and built-in tools.
- Handoffs: intelligently transfer control between agents.
- Guardrails: configurable safety checks for input and output validation.
- Tracing & Observability: visualize agent execution traces to debug and optimize performance.

Suitable for customer support automation, multi-step research, content generation, code review, and sales prospecting. Example customers: Coinbase (built AgentKit toolkit for on-chain agents in a few hours), Box (web search + Agents SDK over unstructured enterprise data). The Agents SDK works with the Responses API and Chat Completions API, and with models from other providers that offer a Chat Completions style endpoint. Python first, with Node.js support coming. Design inspired by Pydantic, Griffe, and MkDocs. OpenAI is committed to continuing to build the Agents SDK as an open source framework.

## What's next
OpenAI states agents will become integral to the workforce and commits to providing building blocks for autonomous systems, with deeper API integrations and new tools to deploy, evaluate, and optimize agents in production.
