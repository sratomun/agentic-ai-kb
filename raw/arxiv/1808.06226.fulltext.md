# SentencePiece: A simple and language independent subword tokenizer and detokenizer for Neural Text Processing

Taku Kudo, John Richardson — Google, Inc.

## Abstract
This paper describes SentencePiece, a language-independent subword tokenizer and detokenizer designed for Neural-based text processing, including Neural Machine Translation. It provides open-source C++ and Python implementations for subword units. While existing subword segmentation tools assume that the input is pre-tokenized into word sequences, SentencePiece can train subword models directly from raw sentences, which allows us to make a purely end-to-end and language independent system. We perform a validation experiment of NMT on English-Japanese machine translation, and find that it is possible to achieve comparable accuracy to direct subword training from raw sentences. We also compare the performance of subword training and segmentation with various configurations. SentencePiece is available under the Apache 2 license at https://github.com/google/sentencepiece.

## 1 Introduction
Although NMT can potentially perform end-to-end translation, many NMT systems are still relying on language-dependent pre- and postprocessors used in traditional SMT systems. Moses implements a useful pre/postprocessor, but it is built upon hand-crafted and language-dependent rules whose effectiveness for NMT has not been proven, and is mainly designed for European languages where words are segmented with whitespaces. To train NMT for non-segmented languages such as Chinese, Korean and Japanese, word segmenters must be run independently. SentencePiece implements two subword segmentation algorithms: byte-pair-encoding (BPE) (Sennrich et al., 2016) and unigram language model (Kudo, 2018), with the extension of direct training from raw sentences.

## 2 System Overview
SentencePiece comprises four main components: Normalizer, Trainer, Encoder, and Decoder. Normalizer normalizes semantically-equivalent Unicode characters into canonical forms. Trainer trains the subword segmentation model from the normalized corpus. Encoder normalizes the input and tokenizes it into a subword sequence. Decoder converts the subword sequence back into normalized text.

## 3 Library Design
### 3.1 Lossless Tokenization
Raw text and tokenized sequence are usually not reversibly convertible (e.g. the information that no space exists between "world" and "." is lost). SentencePiece implements the Decoder as an inverse operation of Encoder: Decode(Encode(Normalize(text))) = Normalize(text). This is called lossless tokenization. The basic idea is to treat the input text just as a sequence of Unicode characters; even whitespace is handled as a normal symbol, escaped with a meta symbol _ (U+2581). E.g. "Hello_world." → [Hello] [_wor] [ld] [.], detokenized via ''.join(tokens).replace('_', ' '). subword-nmt uses @@ as an intra-word boundary marker, which cannot always perform lossless tokenization (consecutive whitespaces cannot be encoded).

### 3.2 Efficient subword training and segmentation
Existing subword tools train from pre-tokenized sentences. SentencePiece employs speed-ups: given an input of length N, naive BPE requires O(N^2); SentencePiece adopts an O(N log(N)) algorithm with merged symbols managed by a binary heap (priority queue). Training and segmentation complexities of unigram language models are linear to input data size.

### 3.3 Vocabulary id management
SentencePiece specifies the final size of vocabulary (--vocab_size) rather than the number of merge operations. It reserves ids for special meta symbols (<unk>, <s>, </s>, <pad>) and supports custom meta symbols (e.g. <2ja>, <2de> for multilingual models).

### 3.4 Customizable character normalization
By default normalizes with Unicode NFKC. Normalization implemented as string-to-string mapping with leftmost longest matching, compiled into a finite state transducer (Aho-Corasick automaton). Custom rules can be defined as a TSV file.

### 3.5 Self-contained models
The SentencePiece model file includes the vocabulary, segmentation parameters, AND the pre-compiled finite state transducer for character normalization. Behavior is determined only by the model file with no external dependencies, guaranteeing perfect reproducibility. Stored as a binary Protocol buffer.

### 3.6 Library API for on-the-fly processing
Provides C++, Python and TensorFlow library APIs for on-the-fly processing, enabling sub-sentence level data augmentation and noise injection (e.g. subword regularization (Kudo, 2018) which randomly changes subword segmentation during training; example: "New York" tokenized differently on each SampleEncodeAsPieces call).

Subword segmentations with SentencePiece consistently improve BLEU over the word model (consistent with Sennrich et al. 2016). Pre-tokenization is not always necessary; in English→Japanese the BLEU score is degraded with pre-tokenization. Larger improvements when SentencePiece is applied to Japanese (a non-segmented language) and when the target is Japanese.

## 5 Conclusions
SentencePiece is an open-source subword tokenizer and detokenizer for Neural-based text processing that directly converts text into an id sequence, enabling a purely end-to-end system without language-specific resources. The model file is self-contained to guarantee perfect reproducibility of normalization and subword segmentation.
