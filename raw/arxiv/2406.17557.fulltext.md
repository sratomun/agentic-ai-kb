# Full-text capture — arXiv:2406.17557 "The FineWeb Datasets: Decanting the Web for the Finest Text Data at Scale" (via alphaXiv analysis, ingested 2026-06-22)

Authors: Guilherme Penedo, Hynek Kydlíček, Loubna Ben Allal, Anton Lozhkov, Margaret Mitchell, Colin Raffel, Leandro Von Werra, Thomas Wolf (Hugging Face).

## Objectives
Minimize the "data gap" between proprietary and public LLM pretraining datasets; release FineWeb (general-purpose web text) and FineWeb-Edu (educational subset); document and ablate curation choices; release the datatrove codebase and trained ablation models.

## Experimental Setup
- **Models:** 1.71B-parameter Llama-architecture ablation models, sequence length 2048, GPT-2 tokenizer.
- **Training scale:** filtering ablations on ~28B tokens; some dedup experiments and cumulative checks on 350B tokens. Total effort: over 70 models, ~80,000 H100 GPU hours.
- **Evaluation (lighteval):** CommonSense QA, HellaSwag, OpenBook QA, PIQA, SIQA, WinoGrande, ARC, MMLU.

## Pipeline Stages
1. **Text extraction:** trafilatura on WARC files beat default WET text (less boilerplate, better model performance). Source: 96 Common Crawl snapshots.
2. **Base filtering:** URL blocklist (adult content), fastText language ID (English, score >= 0.65), MassiveText quality/repetition filters → ~**36 trillion tokens**.
3. **Deduplication (MinHash):** 5-grams, 112 hash functions, 14 buckets of 8 hashes, targeting >=75% similarity.
   - **Global deduplication** across all 96 snapshots → **4 trillion tokens**; showed little improvement and sometimes harmed performance (disproportionately removed higher-quality data from older crawls).
   - **Individual (per-snapshot) deduplication** → **20 trillion tokens**; matched RefinedWeb and avoided global-dedup pitfalls. Chosen.
4. **C4 filters:** applied all C4 heuristic filters EXCEPT the terminal-punctuation filter (which removed ~30% of data). Improved performance.
5. **Custom heuristic filters** (developed by comparing high-quality individually-deduplicated vs low-quality globally-deduplicated subsets; 16 candidates, 3 chosen):
   - Fraction of lines ending with punctuation <= 0.12 (removed 10.14% tokens)
   - Fraction of characters in duplicated lines >= 0.1 (removed 12.47% tokens)
   - Fraction of lines shorter than 30 characters >= 0.67 (removed 3.73% tokens)
6. **FineWeb-Edu (educational filtering):** Llama-3-70B-Instruct scored 460,000 webpages (CC-MAIN-2024-10) for educational quality 0–5; a linear-regression classifier on Snowflake-arctic-embed-m embeddings was trained on those annotations; threshold of 3 chosen. This step required 6,000 H100 GPU hours.
   PII removal (email + public IP anonymization) applied for release.

## Main Findings
- **FineWeb** = **15-trillion-token** dataset. Each processing step gave a relative performance boost; FineWeb beats other open web-scale datasets (RefinedWeb, C4, Dolma, The Pile, SlimPajama, RedPajama2, etc.).
- **Deduplication insight:** individual per-snapshot MinHash significantly outperformed global deduplication — challenging the "more aggressive global dedup is always better" assumption.
- **FineWeb-Edu** = **1.3-trillion-token** subset. Dramatically better on knowledge/reasoning benchmarks: MMLU rose from **33% to 37%** (12% relative), ARC rose from **46% to 57%** (24% improvement) vs base FineWeb. Achieved MMLU accuracy similar to other large public datasets (e.g., Matrix) with almost **10x fewer training tokens**.
- **Domain fit (Paloma perplexity):** FineWeb fits broad web/social-media sources better; FineWeb-Edu fits academic content, Wikipedia, and programming content better.
- **Bias analysis:** FineWeb overrepresents hegemonic terms ('man' > 'woman'/'non-binary'; 'christian' > other religions); FineWeb-Edu's educational filtering shifts associations toward less intimacy-tied content.

## Significance
Democratizes LLM training; transparent/reproducible data curation; novel systematic heuristic-filter development; LLM-based educational filtering as a scalable quality paradigm; datatrove codebase and ablation models released.
