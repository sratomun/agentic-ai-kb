# Full-text capture — arXiv:2305.10429 "DoReMi: Optimizing Data Mixtures Speeds Up Language Model Pretraining" (via alphaXiv analysis, ingested 2026-06-22)

Authors: Sang Michael Xie, Hieu Pham, Xuanyi Dong, Nan Du, Hanxiao Liu, Yifeng Lu, Percy Liang, Quoc V. Le, Tengyu Ma, Adams Wei Yu (Google DeepMind; Stanford).

## Motivation
Existing approaches to setting domain weights in pretraining mixtures rely on heuristics (e.g., The Pile's default weights) or expensive downstream-task tuning (e.g., PaLM, GLaM). DoReMi (Domain Reweighting with Minimax Optimization) determines optimal domain weights WITHOUT downstream task knowledge, using a small proxy model.

## Method (three steps)
1. **Reference model:** train a small LM (typically **280M params**) with initial baseline domain weights.
2. **Proxy model with Group DRO:** train a same-size proxy model to minimize the worst-case *excess loss* (proxy loss minus reference loss) across domains. The Group DRO optimizer dynamically upweights domains with higher excess loss via exponentiated gradient ascent; the final domain weights are the average over the training trajectory. The robust proxy model is discarded — only the domain weights are kept.
3. **Train large model** with the new domain weights (e.g., an **8B-parameter** main model).

**Iterated DoReMi:** repeat rounds, each using the previous round's weights as the new reference; converges in a few rounds.

**Overhead:** For an 8B main model, training the two 280M models (reference + proxy) is only **8% of total FLOPs**.

## Main Findings
**On The Pile:**
- DoReMi (280M proxy → 8B main) improves average few-shot downstream accuracy by **6.5 percentage points** over The Pile's default heuristic weights.
- Reaches the baseline's accuracy with **2.6x fewer training steps**.
- Reduces perplexity across **all 22 domains** of The Pile despite downweighting some domains (a Pareto improvement) — hypothesized to reallocate samples toward "medium-entropy" domains.
- Reweighting heavily increases Pile-CC (web text) and decreases ArXiv, PubMed Central, StackExchange.

**On GLaM:**
- With no downstream-task knowledge, DoReMi matches the "oracle" downstream-tuned domain weights. Iterated DoReMi converged within 3 rounds.

**Across scales:**
- Same-size proxy/main (280M→280M, 1B→1B): DoReMi improves downstream accuracy by 2% and reaches baseline accuracy **4x faster** on average.
- The produced domain weights help the main model even when the proxy itself doesn't beat its baseline (the output is the robust data mixture, not the proxy model).
- Increasing proxy size from 70M to 280M improved the 8B main model; a 1B proxy did not further improve.

## Significance
A principled, efficient, task-agnostic method for optimizing pretraining data mixtures. Novel reinterpretation of DRO — robust *data distribution* rather than robust *model*. Demonstrates cross-scale transfer (280M proxy → 8B main, ~30x larger). Reduces training cost/time and has fairness implications via worst-case-domain optimization.
