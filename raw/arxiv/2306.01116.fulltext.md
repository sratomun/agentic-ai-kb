# Full-text capture — arXiv:2306.01116 "The RefinedWeb Dataset for Falcon LLM: Outperforming Curated Corpora with Web Data, and Web Data Only" (via alphaXiv, ingested 2026-06-22)

Authors: Penedo, Malartic, Hesslow, Cojocaru, Cappelli, Alobeidli, Pannier, Almazrouei, Launay (LightOn / TII, Falcon LLM team).

## Introduction
RefinedWeb challenges the assumption that high-quality curated corpora are essential, demonstrating that carefully processed web data alone can produce models that match or exceed those trained on curated datasets. It addresses concerns about data bottlenecks in scaling LLMs.

## The Macrodata Refinement (MDR) Pipeline
Three main stages:
1. **Document Preparation**: URL filtering using a blocklist and scoring mechanism; text extraction with Trafilatura; language identification using fastText.
2. **Filtering**: removal of repeated sequences within documents; document-wise quality filtering using heuristics; line-wise corrections.
3. **Deduplication**: fuzzy deduplication using MinHash; exact substring deduplication using suffix arrays; URL-based deduplication across CommonCrawl dumps.

Beginning with raw CommonCrawl, the pipeline progressively refines the dataset, ultimately retaining only about **11.67%** of the original data in the final RefinedWeb dataset.

## Dataset Characteristics
RefinedWeb comprises approximately **5 trillion tokens** of English text derived from CommonCrawl. The authors publicly released a **600 billion token** extract. Most-represented domains are blogspot.com and wordpress.com. Toxicity analysis shows RefinedWeb has toxicity levels comparable to other commonly used datasets such as The Pile and C4.

## Model Training and Architecture
Autoregressive decoder-only LMs of **1B, 3B, and 7B parameters** trained on RefinedWeb, configured similarly to GPT-3 with modern improvements: ALiBi positional encodings, FlashAttention, multi-query attention. Focus on zero-shot generalization, evaluated with the EleutherAI evaluation harness across task aggregates (small, core, main, ext).

## Performance Results
Models trained on RefinedWeb significantly outperform those trained on The Pile and achieve performance comparable to the GPT-3 series despite using only web data. The 7B parameter model trained on RefinedWeb achieves performance comparable to GPT-3 on the core task aggregate, despite no curated data sources — challenging the prevailing assumption that curated corpora are essential.

## Impact of Filtering and Deduplication (Ablations)
- **Filtering**: Models trained on filtered data (RW-F) consistently outperform those on unfiltered data (RW-Raw), with the gap increasing as training progresses.
- **Deduplication**: Consistently improves performance across compute budgets; removing redundant content makes training more efficient and improves performance (reduces memorization of specific text patterns).
- Applying the MDR deduplication stage to other datasets (OSCAR, C4) yielded consistent performance improvements, suggesting broad applicability.

## Significance
Reduced reliance on curated data; addresses data bottlenecks in LLM scaling; highlights the role of data processing (filtering and deduplication); public release of a 600B-token extract and open models. Used to train Falcon-40B.
