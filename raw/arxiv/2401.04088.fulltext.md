## Abstract
We introduce Mixtral 8x7B, a Sparse Mixture of Experts (SMoE) language model. Mixtral has the same architecture as Mistral 7B, with the difference that each layer is composed of 8 feedforward blocks (i.e. experts). For every token, at each layer, a router network selects two experts to process the current state and combine their outputs. Even though each token only sees two experts, the selected experts can be different at each timestep. As a result, each token has access to 47B parameters, but only uses 13B active parameters during inference. Mixtral was trained with a context size of 32k tokens and it outperforms or matches Llama 2 70B and GPT-3.5 across all evaluated benchmarks. In particular, Mixtral vastly outperforms Llama 2 70B on mathematics, code generation, and multilingual benchmarks. We also provide a model fine-tuned to follow instructions, Mixtral 8x7B – Instruct, that surpasses GPT-3.5 Turbo, Claude-2.1, Gemini Pro, and Llama 2 70B – chat model on human benchmarks. Both the base and instruct models are released under the Apache 2.0 license. Code: https://github.com/mistralai/mistral-src Webpage: https://mistral.ai/news/mixtral-of-experts/

## 1 Introduction
Mixtral 8x7B is a sparse mixture of experts model (SMoE) with open weights, licensed under Apache 2.0. Mixtral outperforms Llama 2 70B and GPT-3.5 on most benchmarks. As it only uses a subset of its parameters for every token, Mixtral allows faster inference speed at low batch-sizes, and higher throughput at large batch-sizes. It is a decoder-only model where the feedforward block picks from a set of 8 distinct groups of parameters. At every layer, for every token, a router network chooses two of these groups (the "experts") to process the token and combine their output additively. This technique increases the number of parameters of a model while controlling cost and latency, as the model only uses a fraction of the total set of parameters per token. Mixtral is pretrained with multilingual data using a context size of 32k tokens.

Mixtral 8x7B – Instruct is a chat model fine-tuned to follow instructions using supervised fine-tuning and Direct Preference Optimization (DPO). Its performance surpasses GPT-3.5 Turbo, Claude-2.1, Gemini Pro, and Llama 2 70B – chat model on human evaluation benchmarks. Submitted changes to vLLM (integrates Megablocks CUDA kernels). Skypilot allows deployment of vLLM endpoints on cloud.

## 2 Architectural details
Model architecture (Table 1): dim 4096; n_layers 32; head_dim 128; hidden_dim 14336; n_heads 32; n_kv_heads 8; context_len 32768; vocab_size 32000; num_experts 8; top_k_experts 2.

Mixtral is based on a transformer architecture and uses the same modifications as Mistral 7B, except Mixtral supports a fully dense context length of 32k tokens, and the feedforward blocks are replaced by Mixture-of-Expert layers.

### 2.1 Sparse Mixture of Experts
Given n expert networks {E_0, ..., E_{n-1}}, the output of the expert layer is sum_i G(x)_i · E_i(x). A simple and performant gating is G(x) := Softmax(TopK(x · W_g)), where TopK keeps the top-K logits and sets the rest to −∞. K (number of experts used per token) is a hyper-parameter modulating compute per token. Increasing n while keeping K fixed increases the model's parameter count while keeping its computational cost effectively constant — distinguishing the model's total (sparse) parameter count, which grows with n, from the active parameter count, which grows with K.

MoE layers run efficiently on single GPUs with specialized kernels (e.g. Megablocks casts FFN operations of the MoE layer as large sparse matrix multiplications). Can be distributed via Model Parallelism and Expert Parallelism (EP). For Mixtral, the same SwiGLU architecture is used as the expert function E_i(x) and K=2. Output: y = sum_i Softmax(Top2(x·W_g))_i · SwiGLU_i(x). Similar to GShard architecture except Mixtral replaces all FFN sub-blocks by MoE layers (GShard replaces every other block) and GShard uses a more elaborate gating strategy for the second expert.

### 3.1 Multilingual benchmarks
Mixtral significantly outperforms Llama 2 70B in French, German, Spanish, and Italian (Table 4).

### 3.2 Long range performance
Mixtral achieves 100% retrieval accuracy on the passkey retrieval task regardless of context length (up to 32k) or passkey position. Perplexity on proof-pile decreases monotonically as context size increases.

### 3.3 Bias Benchmarks
BBQ accuracy: Llama 2 70B 51.5% vs Mixtral 56.0%. BOLD sentiment: Mixtral displays more positive sentiments than Llama 2 with similar variances.

## 4 Instruction Fine-tuning
Mixtral – Instruct trained via SFT then DPO. Reaches 8.30 on MT-Bench, best open-weights model as of December 2023. LMSys: Mixtral 8x7B Instruct v0.1 Arena Elo 1121, outperforming Claude-2.1 (1117), GPT-3.5-Turbo (1117 best), Gemini Pro (1111), Llama-2-70b-chat (1077).

## 5 Routing analysis
Measured distribution of selected experts on subsets of The Pile. No obvious patterns in expert assignment based on topic (ArXiv, biology/PubMed, Philosophy similar). Only DM Mathematics marginally different. Router exhibits structured syntactic behavior and positional/temporal locality: e.g. 'self' in Python and 'Question' in English often routed through the same expert; indentation tokens assigned to the same experts. Table 5: percentage of consecutive same-expert assignment higher than random (random = 12.5% first choice, ~46% first-or-second) at layers 15 and 31.

## 6 Conclusion
Mixtral 8x7B is the first mixture-of-experts network to reach state-of-the-art performance among open-source models. Uses 13B active parameters per token while outperforming Llama 2 70B (70B params per token). Released under Apache 2.0.
