# XFlow: An Executable Protocol Programming System for Reliable Multi-Agent Workflows

**Why it matters:** Tackles the underspecified prompt↔harness boundary — which workflow commitments belong in prompts vs enforced harness structure — with a compiled protocol language. Directly relevant to building reliable agent workflows.

## Takeaways
- XFlow + XPF: a literate-but-compiled protocol language that moves selected workflow commitments out of prompts into checkable harness structure.
- Runtime stages uncertainty through lifecycle-governed typed symbols (state cells with validation/commit states); actor outputs are mediated before becoming shared state.
- Evaluated on constrained interaction, long-context reasoning, and agentic software engineering — reliability gains from making constraints explicit and enforceable.
