# Vibe engineering

*[[simon-willison]] names the *responsible* end of the spectrum — professionals accelerating with [[coding-agents]] while staying accountable. The counterpart to [[vibe-coding]].*

## What he argues
- **A terminology gap needed filling.** If [[vibe-coding]] is the "fast, loose, irresponsible" end, what's the other end — seasoned engineers using LLMs while "proudly and confidently accountable" for the result? He proposes **vibe engineering** (tongue partly in cheek). *Update (Feb 2026): the field is settling on "Agentic Engineering" instead.*
- **Coding agents raised the bar.** Claude Code (Feb 2025), Codex CLI (Apr), Gemini CLI (Jun) iterate-test-modify in a loop; experienced engineers now run several agents in parallel.
- **LLMs reward classic senior-engineering discipline** — and this is the load-bearing claim: automated testing (test-first lets agents *fly*), upfront planning, comprehensive docs, good Git habits, CI/automation, a code-review culture, "a very weird form of management," strong manual QA, research skills, preview environments, an instinct for what to outsource, and updated estimation.
- **AI amplifies existing expertise.** "The more skills and experience you have… the faster and better the results." The work shifts from writing code to architecture, specs, success criteria, designing agentic loops, and *a lot* of code review.

## Why it's on the radar
The opinion-layer anchor for the distinction between [[vibe-coding]] and disciplined agentic coding, and a bridge to [[harness-engineering]] (the practices here — tests, CI, preview envs, agentic loops — are the human-process half of the harness). Note the live terminology fight: vibe engineering vs "agentic engineering."
