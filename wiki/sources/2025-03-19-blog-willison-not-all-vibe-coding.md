# Not all AI-assisted programming is vibe coding (but vibe coding rocks)

*[[simon-willison]] on the precise meaning of [[vibe-coding]] — and why semantic diffusion is wrecking it. The definitional anchor for the radar's vibe-coding hub.*

## What he argues
- **The original definition is narrow.** Karpathy coined "vibe coding" (Feb 6 2025) for *"fully give in to the vibes… forget that the code even exists"* — Accept All, don't read the diffs, paste errors back without comment. Willison sharpens it: vibe coding means **"building software with an LLM without reviewing the code it writes."**
- **Semantic diffusion is the enemy.** The term is "already escaping its original intent" as people stretch it to cover *all* AI-assisted coding. That both dilutes it and misrepresents responsible practice. If you reviewed, tested, and can explain the code, "that's not vibe coding, it's software development — the usage of an LLM is immaterial."
- **But vibe coding genuinely rocks** for low-stakes work: it shaves the steep learning curve "down to almost flat," democratizing custom-tool building, and is the best way for experienced devs to build intuition for what LLMs can/can't do.
- **Guardrails are situational:** keep it low-stakes, watch for secrets/data privacy, beware usage-billed APIs, prefer sandboxes (praises Claude Artifacts' locked-down iframe).
