# Graph-based Agent Memory: Taxonomy, Techniques, and Applications

**SURVEY** · arXiv [2602.05665v1](https://arxiv.org/abs/2602.05665v1) · 2026-02-05 · Chang Yang, Chuang Zhou, Yilin Xiao, Su Dong et al.

**Why it matters:** Your knowledge-graph interest lives here: it's the best 2026 map of how graph/KG structures serve agent memory and retrieval.

## What it surveys
Agent memory from the graph perspective — the closest thing 2026 has to a knowledge-graph-for-agents survey.

## Framework / taxonomy
Taxonomy across short/long-term, knowledge vs experience, non-structural vs structural memory, with a graph implementation view; analyzes the memory life cycle (extraction → storage → retrieval → evolution) and catalogs open-source libraries/benchmarks.

## Notable points
- Argues graphs are uniquely suited to model relational dependencies, hierarchy, and efficient retrieval for memory.
- Covers self-evolving memory and its evaluation resources (curated repo).
- Directly relevant to GraphRAG and ontology-backed agents.
