# Agentic AIBOM

*AI Bill of Materials extended from static SBOMs into active, agent-generated provenance artifacts — capturing runtime behavior, environment drift, and exploitability context.*

## What it is
Agentic AIBOM extends traditional software Bill of Materials concepts into live, runtime provenance artifacts generated by a multi-agent architecture ([[mcp]] + [[a2a-protocol|A2A]]). Unlike static SBOMs, it captures what an agent *did* at runtime — behaviors, environment drift, and exploitability context — using CSAF/VEX semantics.

## Why it matters
Static SBOMs don't capture what an agent *did* — only what it was made of. Enterprise compliance teams and auditors will eventually require runtime provenance for agentic pipelines, particularly in regulated industries. Agentic AIBOM is the first framework proposing how to produce it.
